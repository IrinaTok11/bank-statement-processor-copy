Bank Statement Processor — Python

Automation that consolidates multi-bank statements, categorizes transactions, applies forensic screening rules, and generates audit-ready findings — deterministic, repeatable analysis every run

Repository Quick start

Python automation summary

This isn't a static template or a collection of Excel macros. It's a forensic-grade Python pipeline that processes real bank statement exports from multiple institutions: the script auto-detects formats, standardizes transactions into a canonical schema, applies keyword-based categorization (20+ operation types), runs explainable screening rules (duplicate detection, split payments, vendor concentration growth, round-amount clustering, weekend activity, new counterparty alerts), and outputs a five-sheet workbook with full source traceability — Transactions, Findings & Alerts, Evidence Index, Analytics Dashboard, Config & Thresholds. The methodology shown here has been developed and refined through practical bankruptcy trustee work, adapted for demonstration with synthetic data.

Why this project is different

  • Multi-format intelligence: Auto-detects Chase, Santander, ABSA formats plus generic patterns — no manual configuration files
  • Explainable categorization: Keyword-based rules (transparent, auditable) instead of ML black boxes — analysts can verify why transactions were classified
  • Forensic-grade screening: Six detection algorithms (duplicates, split payments, vendor concentration, round amounts, weekend activity, new counterparties) with documented thresholds
  • Full audit traceability: Every transaction links back to source file/sheet/row — critical for legal proceedings and compliance reviews
  • Deterministic pipeline: Same inputs → same outputs (no randomness, no ML variability) — reproducible for court exhibits
  • Five-sheet reporting: Transactions + Findings + Evidence + Analytics + Config — comprehensive analysis in one workbook

Real-world applications

Bankruptcy proceedings

Identifying suspicious transactions in the twilight period before insolvency — preferential payments, undervalue transfers, and asset dissipation schemes. The processor's source traceability enables court-ready exhibits showing exact statement references.

Fraud investigations

Detecting duplicate invoicing, split payment schemes to circumvent approval thresholds, vendor concentration anomalies suggesting kickback arrangements, and weekend disbursements outside normal business patterns.

Vendor risk management

Tracking concentration growth (>2x spend increase), identifying new high-value counterparties (>$50k first transactions), and monitoring payment patterns for procurement policy compliance.

Financial statement prep

Consolidating accounts from multiple banks, categorizing transactions for expense reporting, and generating cash flow summaries with vendor concentration analysis — all with Excel/Power BI compatibility.

Limitations and not a fit

  • Requires Excel statement exports (.xlsx/.xls) — PDF statements need conversion
  • English transaction descriptions — keyword rules optimized for Latin script
  • Categorization achieves ~85-95% accuracy — edge cases require manual review
  • Large datasets (>10,000 transactions) may take several minutes — consider splitting by period
  • Duplicate detection uses vendor+amount+time window — doesn't account for legitimate recurring payments

Unique value proposition

1. Time savings: 40+ hours → 5 minutes per analysis

Manual process baseline: Forensic analysts typically spend 2-5 days per case on statement consolidation and preliminary screening — opening multiple Excel files, standardizing formats, manually flagging duplicates, tracking vendor patterns, cross-referencing transactions.

Automated workflow: This processor reduces that to a 5-minute script run. The analyst's focus shifts from data wrangling to high-value interpretation of findings — investigating P1 alerts, interviewing management, gathering supporting evidence.

Scale impact: For professionals handling multiple cases annually, the time savings compound significantly — equivalent to freeing up capacity for additional casework or deeper analysis.

2. Consistency and auditability

Problem: Manual Excel analysis is prone to human error — missed duplicates, inconsistent categorization, lost source references, unreproducible results when opposing counsel challenges findings.

Solution: Deterministic pipeline with documented rules. Same inputs always produce same outputs. Config & Thresholds sheet documents every threshold and rationale. Evidence Index provides exact source file/sheet/row for court verification.

Court defensibility: "Your Honor, Alert A0001 references Transaction TXN_00409 found in GLOBAL_BUSINESS_BANK_USD.xlsx, sheet GBB_BizChk_USD_2023-07, row 22" — immediate verification possible.

3. Scalability: from 3 statements to 50+ accounts

Typical scenario: Large bankruptcies involve companies with 10-20 bank accounts across multiple institutions and currencies. Manual consolidation becomes prohibitively time-consuming.

Processor capability: Handles 50+ statement files in one run. Auto-detects formats. Generates unified Transactions table with Bank column for filtering. Analytics Dashboard shows cross-account patterns.

International cases: Processes USD, GBP, EUR statements (assuming English descriptions) — currency conversion can be added as needed.

4. Forensic depth: beyond basic duplicate detection

Standard tools: Most accounting software offers basic duplicate detection (exact match only) and simple expense reports.

This processor provides:

  • Fuzzy duplicate detection — ±1% tolerance catches invoice rounding variations
  • Split payment patterns — detects approval limit circumvention schemes
  • Vendor concentration growth — flags rapidly increasing dependency (>2x baseline spend)
  • Benford's Law analysis — statistical anomaly detection for fabricated transactions
  • Weekend activity screening — unusual non-business-hours disbursements
  • New counterparty alerts — first-time large payments requiring enhanced due diligence

5. Knowledge transfer and training

Challenge: Junior analysts need 6-12 months to develop forensic intuition for statement analysis — what patterns matter, where to look, how to prioritize findings.

Processor as teacher: The Findings & Alerts sheet codifies senior practitioner expertise. Junior analysts learn by reviewing flagged transactions and understanding why they matter (methodology documented in Config & Thresholds).

Team scaling: Reduces reliance on scarce senior forensic expertise. Junior analysts can handle preliminary screening independently.

6. Client value and competitive differentiation

For professional services firms: Faster turnaround on client deliverables. Higher-quality work product (consistent categorization, comprehensive screening). More cases handled per analyst.

For corporate clients: Proactive fraud monitoring. Vendor risk management. Compliance documentation. Audit trail for regulatory reviews.

Market positioning: "We use proprietary forensic automation to deliver 3x faster analysis with audit-grade traceability" — tangible competitive advantage.

Tech stack and run

  • Python: 3.10+
  • Libraries: pandas (data processing), openpyxl (Excel I/O), numpy (numerical operations)
  • Architecture: Single-file script with modular functions (format detection, normalization, categorization, screening, reporting)
  • Data: Synthetic statement examples for demonstration; output workbook with five analysis sheets
  • Deployment: Static GitHub Pages for documentation; Python script runs locally

How to run

Shell 
cd run
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
# Install & run:
pip install -r ../requirements.txt
python bank_statement_processor.py

Result: The script reads all .xlsx/.xls files in run/, consolidates them, applies forensic rules, and creates consolidated_statements.xlsx in run/ with five analysis sheets.
